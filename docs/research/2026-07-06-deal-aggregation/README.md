## 1. 一句話結論

**ShareGood 不需要重構，需要的是「補完主動線＋長出資訊型好康」**：M0–M8 已蓋好新願景約六成的地基（優惠券加密揭露、到期 job、訂閱通知、治理後台、資料權利、稽核營運），真正的落差集中在（a）首頁與搜尋頁根本不存在（主動線斷裂）、（b）「資訊型好康」這個新內容類型、（c）失效回報與來源標示機制、（d）票券/點數的法務邊界。

## 2. 現況摘要（詳見 01）

- **已具備**：實體物品＋優惠券主迴路、AES-256-GCM 券碼加密＋領取後揭露＋揭露稽核、到期自動下架 job、關鍵字/分類/縣市訂閱＋Web Push、檢舉/強制下架/功能限制/申訴/rate limit/關鍵字黑名單、資料匯出/帳號刪除/legal hold/司法調閱、稽核/健康檢查/備份 runbook、抽籤防秒搶、條款/隱私完整草案（掛 AI 起草警語）、圖片 EXIF 自動移除。

## 4. 關鍵研究結論（詳見 02/03/04）

1. **自動化爬蟲聚合＝高風險，排除於所有方案之外**：Lawsnote 案一審（2025）將「違反 ToS 的爬取」連結刑法 359 條、民事判賠約 1.05 億（上訴中）；且實查 18 個來源後確認**沒有任何台灣速食/超商/量販提供優惠資料 API 或 RSS**，官方頁多為 JS SPA 且部分明文禁轉載（麥當勞、肯德基）或有反自動化措施（摩斯、OPEN POINT 403）。合法路徑只剩：使用者投稿（S3）＋編輯人工收錄自寫摘要＋導流（S1）＋政府開放資料（S0，超商門市清單）。
2. **票券類型「有條件可做」**：無償轉贈在黃牛條款（文創法 10-1、運動條例 24-1）文義之外——運動條例甚至連「免費票拿來賣」都罰，反而確立無償轉贈是唯一安全區。但需配套：無金額欄位、加價關鍵字攔截、轉贈紀錄保留、法定警示文案；「平台媒介加價的幫助責任」查無依據，需律師。**LINE 即享券官方明文禁轉贈→不可上架清單；7-11/全家寄杯類為官方閉環→只引導官方功能**。我們的利基＝官方機制外的券（紙本券、序號券、跨品牌券）。
3. **點數類型：查無可靠法規依據，未經律師查證前不做**。
4. **市場可借鏡 Top 3**：失效多人回報＋可逆狀態（hotukdeals）、give-to-get 領取配額掛貢獻值（CouponShare）、「不讓人白跑」為文案與功能主軸（台灣贈券社群核心痛點語彙）。全部可掛在既有基礎設施上（M2 檢舉模式、M1 貢獻值、M3 到期 job）。
